Реализация поставленных ведущим курса задач решена с помощью PDO.

В этом варианте, задачи функций (внутри папки models) реализованы внутри файлов index.
Так сделано по 2 причинам:
1 - было изначально интересно сделать это не через дебри объяснений ведущего, а так как "я знаю"
и попрактиковать чуть ранее изученное ещё раз, для закрепления;
2 - на этапе реализации функции редактирования через отдельный файл с функциями (в папке models)
система не смогла осуществить перезапись данных после выдачи имеющейся статьи в веб-форме (функция редактирования).
Итак, реализация предполагает использование PDO - Расширение Объекты данных PHP.
Собственно, почему так - в приведенных ниже ссылках рассказывается о PDO, и довольно подробно, поможет разобраться в коде:
 - Определение: http://php.net/manual/ru/intro.pdo.php
 - Что это за PDO и почему оно есть хорошо(Хабр): https://habrahabr.ru/post/137664/
-------------------------
Итак, реализация предполагает использование файла index.php для основного, "серверного" кода, файлов вида ххх.html.php для отображения
файлов разметки и форм, что в редакторе сильно помогает сориентироваться во время работы, ну и соответственно ускорить дело.
Таким, образом имеем 2 папки - admin с файлом index.php для админа и views для файлов разметки-передачи значений,
которые потому и имеют вид .html.php - разметки много, php-кода мало.
Файл database - отдельный, для подключения к базе данных. Данные соответственно там вписаны мои.
!!! В базе данных в таблице Столбец с текстами статей идет под именем 'text', а не 'content', как у ведущего - тут может быть ошибка.
Участок $_SERVER['DOCUMENT_ROOT'] в строке подключения к БД в начале .php-файлов служит для устранения
неудобства вида "C:/Documents/.../мой сайт(проект, etc)" и посылает сразу в вашу папку, от которой начинается ваш сервер
(в моем случае папка "Course" - это корневая папка.). Тут сами подставляйте свои значения.
Файл output.html.php - отображение сообщения о подключении/ошибке работы с базой данных.
Файл article - был создан по ходу изучения курса с ведущим, и как-то остался. Может через него будет проще понять код, там
его поменьше.
-------------------------
От предлагаемых ведущим стилей отказался, написал свои, просто для удобства чтения. Могут остаться старые имена стилей(но вроде нет).
Проверка админа удалена - вход свободный.
При подключении файла БД на каждой странице вылетает сообщение "Подключение выполнено". Редактируется в файле database.php
Также, имеется проверка удаления статьи. При необходимости можно удалить.
